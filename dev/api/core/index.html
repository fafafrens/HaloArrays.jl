Arrays, layout & reductions

Construction, ownership/layout accessors, indexing, reductions, thread backends, gather, and HDF5 output.

HaloArrays.LocalMultiHaloArray — Method
LocalMultiHaloArray(T, owned_dims, halo; boundary_conditions)

A MultiHaloArray whose fields are LocalHaloArrays (single-process). boundary_conditions is a NamedTuple of per-field boundary conditions, which also fixes the field names. See MultiHaloArray.

source
HaloArrays.ThreadedMultiHaloArray — Method
ThreadedMultiHaloArray(T, tile_size, halo; dims, boundary_conditions)

A MultiHaloArray whose fields are ThreadedHaloArrays sharing one tile layout (dims). synchronize_halo! exchanges every field's tiles in a single call. boundary_conditions is a NamedTuple of per-field boundary conditions. See MultiHaloArray and ThreadedHaloArray.

HaloArrays.boundary_condition_threads! — Function
halo_exchange_threads!(u)
boundary_condition_threads!(u)
synchronize_halo_threads!(u)

Threaded variants of halo_exchange! / boundary_condition! / synchronize_halo! for a ThreadedHaloArray: the per-tile work runs in parallel through the array's thread_backend. The default (non-_threads!) versions are a serial tile loop, which is allocation-free and usually faster for small halo surfaces — reach for these only when benchmarking shows the parallel exchange wins (large surfaces, wide halos, many tiles).

HaloArrays.fill_from_global_indices! — Method
fill_from_global_indices!(f, u)

Set each interior cell from f(I, J, …) evaluated at its global grid index (1-based over the whole domain). Each rank/tile fills only the cells it owns, so the same f produces a consistent global field across MPI ranks and threads — the idiomatic way to set an initial condition. Cf. fill_from_local_indices!.

Example

fill_from_global_indices!(u) do i, j
    exp(-((i - nx/2)^2 + (j - ny/2)^2) / 50)
end
source
HaloArrays.fill_from_local_indices! — Method
fill_from_local_indices!(f, u)

Set each interior cell from f(i, j, …), where the indices are local to this rank's/tile's interior region (1-based, ghost-free). For coordinates on the global grid (the usual choice for initial conditions), use fill_from_global_indices!.

source
HaloArrays.get_recv_view — Method
get_recv_view(side, dim, field) -> view

A mutable view of the halo-wide ghost band on the (side, dim) boundary — the cells a halo exchange (or a boundary condition) writes. Same shape as get_send_view on that side, so a BC can pair them cell-by-cell.

source
HaloArrays.get_send_view — Method
get_send_view(side, dim, field) -> view

A view of the halo-wide band of interior cells adjacent to the (side, dim) boundary — the cells a halo exchange reads, and the interior edge a boundary condition reads. Paired with get_recv_view (same shape) when writing a boundary condition. See also the layout discussion in the source.

source
HaloArrays.global_size — Method
global_size(u) -> dims

Size of the whole grid across all ranks / tiles (interior_size .* topology.dims). For LocalHaloArray it equals interior_size; for HaloArray (MPI) and ThreadedHaloArray it is larger than this rank's/tile's share.

halo_backend(x) -> AbstractHaloBackend

Return a singleton trait describing the storage/execution backend of a halo array or halo collection. Use this for dispatch when code needs separate MPI, local, or threaded implementations while still accepting collection wrappers.

HaloArrays.halo_width — Method
halo_width(u) -> Int

The ghost-cell width on each side (the same in every dimension). Encoded in the type, so it is a compile-time constant. Also callable on the type.

source
HaloArrays.interior_axes — Method
interior_axes(halo[, i])

Axes of the interior (ghost-free) region local to this process. Use axes(halo) for the global logical axes and interior_axes(halo) when looping over data this process can update directly.

source
HaloArrays.interior_range — Method
interior_range(u) -> NTuple of UnitRanges

The index ranges of the interior cells within the padded storage ((halo+1) : (size-halo) per dimension). Use it to index parent(u) in a stencil with ghost-safe offsets, e.g. for I in CartesianIndices(interior_range(u)). For ThreadedHaloArray the range is the same for every tile. See also interior_view.

source
HaloArrays.interior_size — Method
interior_size(halo[, i])

Size of the interior (ghost-free) region local to this process. For HaloArray this is the local MPI subdomain size, not the global distributed size; for serial containers it equals the full logical size.

source
HaloArrays.interior_to_global_index — Method
interior_to_global_index(u::ThreadedHaloArray, tile_id, interior_idx) -> NTuple

Map a tile-local interior index interior_idx — 1-based, excluding the halo — in tile tile_id to its global index in the full domain. Mirrors interior_to_global_index on HaloArray and LocalHaloArray, with the extra tile_id since the interior region is per tile.

source
HaloArrays.interior_view — Method
interior_view(u) -> view

A mutable view of u's interior (ghost-free) cells. The usual way to read/write initial data: interior_view(u) .= .... For ThreadedHaloArray pass a tile id: interior_view(u, tile_id). See also interior_range.

source
HaloArrays.storage_size — Method
storage_size(u[, i]) -> dims

Size of the backing storage including ghost padding (interior + 2*halo per dimension); for ThreadedHaloArray this is the per-tile storage. Contrast with interior_size (ghost-free) and global_size.

HaloArrays.tile_parent — Method
tile_parent(u, tile_id) -> Array

The raw, ghost-padded storage array for tile tile_id. This is what you read and write inside a tile loop, indexed over the shared interior_range(u) (with ghost-safe stencil offsets).

source
HaloArrays.tile_size — Method
tile_size(u) -> dims

Interior (ghost-free) cells per tile, in each dimension. The global interior size is tile_size(u) .* dims (the tile layout).

source
HaloArrays.tile_foreach — Function
tile_foreach(backend, f, itr; scheduler=:dynamic)

Apply f to every element of itr in parallel, according to backend. The scheduler hint is honoured by OhMyThreadsBackend and ignored by the others.

source
HaloArrays.tile_mapreduce — Function
tile_mapreduce(backend, f, op, itr; scheduler=:dynamic)

Parallel mapreduce(f, op, itr) according to backend. itr must be non-empty (in this package there is always at least one tile).

HaloArrays.gather_haloarray — Method
gather_haloarray(halo::HaloArray; root=0) -> Array | nothing

Collect a distributed HaloArray onto the root rank: each rank sends its interior (ghost-free) subdomain and root assembles the full global array. Returns the assembled Array on root and nothing on every other rank.

source
HaloArrays.create_haloarray_output_file — Method
create_haloarray_output_file(filename, dataset_name, halo, num_timesteps) -> (file, dataset)

Open (or create) the HDF5 file filename and return it with a fixed-size dataset shaped to hold num_timesteps snapshots of halo. Write each step with write_haloarray_timestep!; close the returned file when done. Under MPI the file is opened collectively across the array's communicator.

source
HaloArrays.gather_and_append_haloarray! — Method
gather_and_append_haloarray!(filename::String, dataset::String, halo::HaloArray; root::Int=0)

Gather halo on root and append it as a new timestep in dataset.

source
HaloArrays.gather_and_save_haloarray — Method
gather_and_save_haloarray(filename, halo; root=0)

Gather halo onto root and write the assembled global array to the HDF5 file filename as a single snapshot. Distributed arrays write only on root; serial arrays write directly. See gather_and_append_haloarray! to append successive timesteps to one dataset instead.

source
HaloArrays.write_haloarray_timestep! — Method
write_haloarray_timestep!(dset, halo, timestep)

Write the interior (ghost-free) data of halo into the time-resolved dataset dset (from create_haloarray_output_file) at index timestep (0-based). Under MPI each rank writes its own subdomain collectively.

source

is_root(cart::CartesianTopology; root=0)

Return true on the root rank (default 0) of an active topology.

source
HaloArrays.field_storages — Method
field_storages(c::FieldCollection)

The raw padded backing array of every field — parent pushed down to the leaves — in the same container kind as the collection (a NamedTuple for MultiHaloArray, an array for ArrayOfHaloArray). Index these with storage indices (ghost-inclusive), e.g. over interior_range or a FaceRanges sweep. Contrast parent (the field container) and interior_view (ghost-free views).

source
HaloArrays.interior_view — Method
interior_view(c::FieldCollection[, tile_id])

The interior (ghost-free) view of every field, in the same container kind as the collection (a NamedTuple for MultiHaloArray, an array for ArrayOfHaloArray). Pass tile_id for the per-tile interior of a threaded collection.

Construction, ownership/layout accessors, indexing, reductions, thread backends, gather, and HDF5 output.

HaloArrays.LocalMultiHaloArray — Method
LocalMultiHaloArray(T, owned_dims, halo; boundary_conditions)

A MultiHaloArray whose fields are LocalHaloArrays (single-process). boundary_conditions is a NamedTuple of per-field boundary conditions, which also fixes the field names. See MultiHaloArray.

source
HaloArrays.ThreadedMultiHaloArray — Method
ThreadedMultiHaloArray(T, tile_size, halo; dims, boundary_conditions)

A MultiHaloArray whose fields are ThreadedHaloArrays sharing one tile layout (dims). synchronize_halo! exchanges every field's tiles in a single call. boundary_conditions is a NamedTuple of per-field boundary conditions. See MultiHaloArray and ThreadedHaloArray.

HaloArrays.boundary_condition_threads! — Function
halo_exchange_threads!(u)
boundary_condition_threads!(u)
synchronize_halo_threads!(u)

Threaded variants of halo_exchange! / boundary_condition! / synchronize_halo! for a ThreadedHaloArray: the per-tile work runs in parallel through the array's thread_backend. The default (non-_threads!) versions are a serial tile loop, which is allocation-free and usually faster for small halo surfaces — reach for these only when benchmarking shows the parallel exchange wins (large surfaces, wide halos, many tiles).

HaloArrays.fill_from_global_indices! — Method
fill_from_global_indices!(f, u)

Set each owned cell from f(I, J, …) evaluated at its global grid index (1-based over the whole domain). Each rank/tile fills only the cells it owns, so the same f produces a consistent global field across MPI ranks and threads — the idiomatic way to set an initial condition. Cf. fill_from_local_indices!.

Example

fill_from_global_indices!(u) do i, j
    exp(-((i - nx/2)^2 + (j - ny/2)^2) / 50)
end
source
HaloArrays.fill_from_local_indices! — Method
fill_from_local_indices!(f, u)

Set each owned cell from f(i, j, …), where the indices are local to this rank's/tile's owned region (1-based, ghost-free). For coordinates on the global grid (the usual choice for initial conditions), use fill_from_global_indices!.

source
HaloArrays.fill_interior! — Method
fill_interior!(u, value)

Set every owned (ghost-free) cell of u to value. Ghosts are left untouched — call synchronize_halo! afterwards if a stencil will read them.

source
HaloArrays.get_recv_view — Method
get_recv_view(side, dim, field) -> view

A mutable view of the halo-wide ghost band on the (side, dim) boundary — the cells a halo exchange (or a boundary condition) writes. Same shape as get_send_view on that side, so a BC can pair them cell-by-cell.

source
HaloArrays.get_send_view — Method
get_send_view(side, dim, field) -> view

A view of the halo-wide band of interior (owned) cells adjacent to the (side, dim) boundary — the cells a halo exchange reads, and the interior edge a boundary condition reads. Paired with get_recv_view (same shape) when writing a boundary condition. See also the layout discussion in the source.

source
HaloArrays.global_size — Method
global_size(u) -> dims

Size of the whole grid across all ranks / tiles (owned_size .* topology.dims). For LocalHaloArray it equals owned_size; for HaloArray (MPI) and ThreadedHaloArray it is larger than this rank's/tile's share.

halo_backend(x) -> AbstractHaloBackend

Return a singleton trait describing the storage/execution backend of a halo array or halo collection. Use this for dispatch when code needs separate MPI, local, or threaded implementations while still accepting collection wrappers.

HaloArrays.halo_width — Method
halo_width(u) -> Int

The ghost-cell width on each side (the same in every dimension). Encoded in the type, so it is a compile-time constant. Also callable on the type.

source
HaloArrays.interior_range — Method
interior_range(u) -> NTuple of UnitRanges

The index ranges of the owned cells within the padded storage ((halo+1) : (size-halo) per dimension). Use it to index parent(u) in a stencil with ghost-safe offsets, e.g. for I in CartesianIndices(interior_range(u)). For ThreadedHaloArray the range is the same for every tile. See also interior_view.

source
HaloArrays.interior_view — Method
interior_view(u) -> view

A mutable view of u's owned (ghost-free) cells. The usual way to read/write initial data: interior_view(u) .= .... For ThreadedHaloArray pass a tile id: interior_view(u, tile_id). See also interior_range.

source

HaloArrays.owned_axes — Method
owned_axes(halo)

Return the owned-cell axes of a halo container on the current process.

Use axes(halo) for the global logical axes and owned_axes(halo) when looping over data that this process can update directly.

source
HaloArrays.owned_size — Method
owned_size(halo)

Return the owned interior size of a halo container on the current process.

For HaloArray this is the owned MPI subdomain size, not the global distributed size. For serial containers it is equal to their full logical interior size.

source
HaloArrays.owned_to_global_index — Method
owned_to_global_index(u::ThreadedHaloArray, tile_id, owned_idx) -> NTuple

Map a tile-local owned (interior) index owned_idx — 1-based, excluding the halo — in tile tile_id to its global index in the full domain. Mirrors owned_to_global_index on HaloArray and LocalHaloArray, with the extra tile_id since the owned region is per tile.

source
HaloArrays.storage_size — Method
storage_size(u[, i]) -> dims

Size of the backing storage including ghost padding (owned + 2*halo per dimension); for ThreadedHaloArray this is the per-tile storage. Contrast with owned_size (ghost-free) and global_size.

HaloArrays.tile_parent — Method
tile_parent(u, tile_id) -> Array

The raw, ghost-padded storage array for tile tile_id. This is what you read and write inside a tile loop, indexed over the shared interior_range(u) (with ghost-safe stencil offsets).

source
HaloArrays.tile_size — Method
tile_size(u) -> dims

Owned (ghost-free) cells per tile, in each dimension. The global owned size is tile_size(u) .* dims (the tile layout).
